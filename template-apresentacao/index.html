Os 4 níveis

Cada nível resolve um problema diferente

  • Nível 1 — Estático: mostra a tela parada. Bom pra crítica visual.
  • Nível 2 — Clicável: simula a navegação. Bom pra fluxo macro.
  • Nível 3 — Interativo: tem estado e responde. Bom pra micro-interação.
  • Nível 4 — Funcional: tem dados reais. É onde a gente vai morar.
Não é sobre largar o Figma. É sobre saber em que ponto ele para de funcionar pro que você precisa projetar.
— Premissa do curso
Estático vs Funcional

Busca com sugestões

● Mockup estático

Você desenha uma tela

3 sugestões hardcoded, todas começando com a mesma letra que o protótipo precisa pra "funcionar" no Figma.

— Não testa edge case
— Não testa loading
— Não testa vazio
— Não testa erro

● Protótipo funcional

Você projeta o comportamento

Digita qualquer letra, a lista filtra de verdade. Apaga tudo, vê o estado vazio. Tira a internet, vê o erro.

— Edge cases aparecem sozinhos
— Loading existe
— Estado vazio é projetado
— Erro é desenhado

Nível 4 · Funcional

Filtro real em 15 linhas

SearchSuggest.jsx 
function SearchSuggest({ items }) {
  const [query, setQuery] = useState('');
  const filtered = items.filter(i =>
    i.toLowerCase().includes(query.toLowerCase())
  );

  return (
    <div>
      <input value={query} onChange={e => setQuery(e.target.value)} />
      {filtered.length === 0 && <Empty />}
      <ul>{filtered.map(i => <li key={i}>{i}</li>)}</ul>
    </div>
  );
}
